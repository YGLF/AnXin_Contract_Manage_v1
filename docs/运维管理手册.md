```bash

  -H "Content-Type: application/json" \
  -d '{

    "password": "password123",
```

## 数据备份与恢复

### 自动备份

建议配置定时任务自动备份：

```bash
# 每天凌晨2点备份
0 2 * * * /path/to/backup.sh
```

### 手动备份

```bash
# 备份数据库
mysqldump -u root -p contract_manage > backup_$(date +%Y%m%d).sql

# 压缩备份
mysqldump -u root -p contract_manage | gzip > backup_$(date +%Y%m%d).sql.gz
```

### 恢复数据

```bash
# 恢复数据
mysql -u root -p contract_manage < backup_20240101.sql

# 从压缩文件恢复
gunzip < backup_20240101.sql.gz | mysql -u root -p contract_manage
```

### 备份策略

| 类型 | 频率 | 保留时间 |
|------|------|----------|
| 全量备份 | 每日 | 30天 |
| 每周备份 | 每周 | 90天 |
| 每月备份 | 每月 | 1年 |

## 日志管理

### 日志位置

| 日志 | 位置 |
|------|------|
| 后端日志 | stdout 或 logs/ |
| MySQL日志 | /var/log/mysql/ |
| 系统日志 | /var/log/syslog |

### 日志轮转

配置 logrotate：

```bash
# /etc/logrotate.d/contract-manage
/path/to/logs/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
}
```

### 查看日志

```bash
# 查看最近100行日志
tail -100 /path/to/logs/app.log

# 实时查看日志
tail -f /path/to/logs/app.log

# 查看错误日志
grep -i error /path/to/logs/app.log
```

## 安全配置

### 防火墙配置

```bash
# 允许SSH
ufw allow ssh

# 允许HTTP/HTTPS
ufw allow 80/tcp
ufw allow 443/tcp

# 允许后端API
ufw allow 8000/tcp

# 启用防火墙
ufw enable
```

### MySQL安全

```bash
# 修改root密码
mysql -u root -p -e "ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';"

# 删除匿名用户
mysql -u root -p -e "DELETE FROM mysql.user WHERE User='';"

# 删除测试数据库
mysql -u root -p -e "DROP DATABASE test;"
```

### JWT配置

修改 `.env` 中的 JWT 配置：

```env
SECRET_KEY=生成的安全密钥
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

生成安全密钥：

```bash
openssl rand -base64 32
```

### SSL配置

生产环境建议配置HTTPS：

1. 获取SSL证书
2. 配置Nginx SSL
3. 重定向HTTP到HTTPS

## 故障处理

### 常见故障

#### 1. 服务无法启动

**症状**：运行 `go run main.go` 失败

**排查步骤**：
1. 检查端口是否被占用：`netstat -tlnp | grep 8000`
2. 检查数据库连接：查看 `.env` 配置
3. 检查日志输出

**解决方法**：
- 杀死占用端口的进程
- 修正数据库配置
- 查看具体错误信息

#### 2. 数据库连接失败

**症状**：Error 1698 Access denied

**排查步骤**：
1. 检查MySQL服务状态
2. 检查用户名密码
3. 检查用户权限

**解决方法**：
```sql
-- 授权
GRANT ALL PRIVILEGES ON contract_manage.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```

#### 3. 前端无法访问后端

**症状**：网络错误/超时

**排查步骤**：
1. 检查后端服务是否运行
2. 检查防火墙规则
3. 检查代理配置

**解决方法**：
- 启动后端服务
- 开放端口：`ufw allow 8000`
- 检查 vite.config.js 代理配置

#### 4. 磁盘空间不足

**症状**：写入失败

**排查步骤**：
1. 检查磁盘使用：`df -h`
2. 查找大文件：`du -sh /*`

**解决方法**：
- 清理日志
- 清理备份文件
- 扩展磁盘

### 应急响应

1. **服务异常**
   - 查看日志确定问题
   - 重启服务
   - 回滚配置

2. **数据异常**
   - 停止服务
   - 恢复备份
   - 检查数据完整性

3. **安全事件**
   - 隔离受影响系统
   - 检查日志
   - 修复漏洞
   - 上报安全事件
