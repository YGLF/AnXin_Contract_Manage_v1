## 开发环境部署

### 1. 克隆项目

```bash
git clone <repository-url>
cd AnXin_Contract_Manage
```

### 2. 安装Go依赖

```bash
go mod download
```

### 3. 配置数据库

#### 安装MySQL

```bash
# Ubuntu
sudo apt update
sudo apt install mysql-server

# 启动MySQL
sudo systemctl start mysql
sudo systemctl enable mysql
```

#### 创建数据库

```bash
mysql -u root -p <<EOF
CREATE DATABASE contract_manage CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EOF
```

### 4. 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```env
APP_NAME=安信合同管理系统
APP_VERSION=1.0.0

MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=contract_manage

SECRET_KEY=your-secret-key-change-in-production
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30

UPLOAD_DIR=uploads

ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin@123456
ADMIN_EMAIL=admin@example.com
```

### 5. 启动后端

```bash
go run main.go
```

后端服务默认在 http://localhost:8000 启动，首次运行会自动创建数据库表。

### 6. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端默认在 http://localhost:3000 启动。

## 生产环境部署

### 1. 准备服务器

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装必要软件
sudo apt install -y git nginx mysql-server
```

### 2. 部署后端

```bash
# 创建用户
sudo useradd -m -s /bin/bash app
sudo mkdir -p /var/www/contract
sudo chown -R app:app /var/www/contract

# 切换到应用用户
sudo su - app
cd /var/www/contract

# 克隆项目
git clone <repository-url> .

# 安装依赖
go mod download

# 配置环境变量
cp .env.example .env
vim .env
```

#### 创建systemd服务

```bash
sudo tee /etc/systemd/system/contract-manage.service <<EOF
[Unit]
Description=Contract Management System
After=network.target mysql.service

[Service]
Type=simple
User=app
WorkingDirectory=/var/www/contract
ExecStart=/usr/local/go/bin/go run main.go
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable contract-manage
sudo systemctl start contract-manage
```

### 3. 部署前端

```bash
cd /var/www/contract/frontend

# 安装依赖
npm install

# 构建生产版本
npm run build

# 配置Nginx
sudo tee /etc/nginx/sites-available/contract <<EOF
server {
    listen 80;
    server_name _;

    root /var/www/contract/frontend/dist;
    index index.html;

    location / {
        try_files \$uri \$uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade \$http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_cache_bypass \$http_upgrade;
    }

    location /uploads {
        alias /var/www/contract/uploads;
    }
}
EOF

# 启用站点
sudo ln -s /etc/nginx/sites-available/contract /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 4. 配置HTTPS（可选）

```bash
# 安装Certbot
sudo apt install certbot python3-certbot-nginx

# 获取SSL证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

## Docker部署

### 1. 安装Docker

```bash
# 安装Docker
curl -fsSL https://get.docker.com | sh

# 启动Docker
sudo systemctl start docker
sudo systemctl enable docker

# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.23.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

### 2. 配置

```bash
cp .env.example .env
```

编辑 `.env` 文件，确保数据库配置正确。

### 3. 启动服务

```bash
docker-compose up -d
```

### 4. 查看状态

```bash
docker-compose ps
```

### 5. 查看日志

```bash
# 所有服务
docker-compose logs -f

# 特定服务
docker-compose logs -f backend
```

### Docker Compose 配置说明

| 服务 | 端口 | 说明 |
|------|------|------|
| mysql | 3306 | MySQL数据库 |
| backend | 8000 | Go后端 |
| frontend | 80 | Vue前端 |

### 数据持久化

数据存储在Docker卷中：
- `mysql_data`: MySQL数据

### 常用命令

```bash
# 启动
docker-compose up -d

# 停止
docker-compose down

# 重启
docker-compose restart

# 重建
docker-compose up -d --build

# 进入容器
docker exec -it contract_backend sh
docker exec -it contract_mysql mysql -u root -p
```

## 高可用部署

### 主从复制

#### 1. 配置主库

编辑 `/etc/mysql/mysql.conf.d/mysqld.cnf`:

```ini
[mysqld]
server-id = 1
log_bin = /var/log/mysql/mysql-bin.log
binlog_do_db = contract_manage
```

#### 2. 配置从库

```ini
[mysqld]
server-id = 2
relay_log = /var/log/mysql/mysql-relay-bin
read_only = 1
```

#### 3. 创建复制用户

```sql
CREATE USER 'repl'@'%' IDENTIFIED BY 'repl_password';
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';
```

### 负载均衡

使用Nginx upstream配置多后端：

```nginx
upstream backend {
    server 127.0.0.1:8000;
    server 127.0.0.1:8001;
    server 127.0.0.1:8002;
}

server {
    location /api {
        proxy_pass http://backend;
    }
}
```

## 验证部署

### 1. 检查服务状态

```bash
# 后端健康检查
curl http://localhost:8000/health

# 前端检查
curl http://localhost/

# 数据库连接
mysql -u root -p -e "SELECT 'OK' as status;"
```

### 2. 功能测试

| 测试项 | 方法 |
|--------|------|
| 用户注册 | POST /api/auth/register |
| 用户登录 | POST /api/auth/login |
| 创建客户 | POST /api/customers |
| 创建合同 | POST /api/contracts |
| 提交审批 | POST /api/contracts/:id/approvals |

### 3. 压力测试

使用 wrk 进行压力测试：

```bash
# 安装wrk
sudo apt install wrk

# 压力测试
wrk -t12 -c400 -d30s http://localhost:8000/health
```
