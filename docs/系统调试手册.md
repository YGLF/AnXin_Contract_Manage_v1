## 常见问题排查

### 1. 后端启动失败

#### 症状
```
panic: Failed to connect database
```

#### 排查步骤

1. **检查MySQL服务**
```bash
systemctl status mysql
ps aux | grep mysql
```

2. **测试数据库连接**
```bash
mysql -h localhost -u root -p
```

3. **检查数据库配置**
```bash
cat .env | grep MYSQL
```

4. **检查端口占用**
```bash
netstat -tlnp | grep 3306
```

#### 解决方案

```sql
-- 授权用户
GRANT ALL PRIVILEGES ON contract_manage.* TO 'root'@'localhost';
FLUSH PRIVILEGES;

-- 或创建专用用户
CREATE USER 'contract_user'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON contract_manage.* TO 'contract_user'@'localhost';
FLUSH PRIVILEGES;
```

---

### 2. 前端无法连接后端

#### 症状
```
Error: Network Error
```

#### 排查步骤

1. **检查后端是否运行**
```bash
curl http://localhost:8000/health
```

2. **检查代理配置**
```bash
cat frontend/vite.config.js
```

3. **检查浏览器控制台**
- 打开Chrome DevTools (F12)
- 查看Console和Network标签

#### 解决方案

检查 `vite.config.js`:

```javascript
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true
      }
    }
  }
})
```

---

### 3. 登录失败

#### 症状
```
Error: invalid username or password
```

#### 排查步骤

1. **检查用户是否存在**
```sql
SELECT id, username, email, role FROM users WHERE username = 'admin';
```

2. **检查密码**
```sql
SELECT username, hashed_password FROM users WHERE username = 'admin';
```

3. **检查JWT配置**
```bash
cat .env | grep SECRET_KEY
```

#### 解决方案

```bash
# 重置密码（需要知道当前密码hash）
# 或通过管理界面重置
```

---

### 4. 数据库连接被拒绝

#### 症状
```
Error 1698 (28000): Access denied for user 'root'@'localhost'
```

#### 原因
MySQL使用auth_socket插件认证

#### 解决方案

```sql
-- 改用密码认证
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_password';
FLUSH PRIVILEGES;
```

---

### 5. WSL连接Windows MySQL

#### 症状
数据库连接超时

#### 排查步骤

1. **获取Windows IP**
```bash
cat /etc/resolv.conf
```

2. **测试连接**
```bash
telnet 172.x.x.x 3306
```

#### 解决方案

在 `.env` 中配置：

```env
MYSQL_HOST=172.x.x.x  # Windows IP
```

## 后端调试

### 启用调试日志

在代码中添加日志：

```go
import "log"

// 调试信息
log.Printf("Debug: %v", variable)

// 结构体
log.Printf("%+v", structVariable)
```

### 使用Delve调试

```bash
# 安装Delve
go install github.com/go-delve/delve/cmd/dlv@latest

# 启动调试
dlv debug main.go

# 设置断点
break main.go:100

# 继续执行
c

# 查看变量
print variableName

# 单步执行
n

# 退出
quit
```

### API调试

```bash
# 健康检查
curl http://localhost:8000/health

# 登录获取token
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin@123456"}'

# 使用token调用API
curl http://localhost:8000/api/contracts \
  -H "Authorization: Bearer <token>"
```

### 运行单元测试

```bash
# 运行所有测试
go test -v ./...

# 运行特定测试
go test -v -run TestUserService ./services/

# 运行集成测试
go test -v -run TestFullApprovalWorkflow ./services/

# 显示覆盖率
go test -v -cover ./...
```

## 数据库调试

### 慢查询分析

```sql
-- 启用慢查询日志
SHOW VARIABLES LIKE 'slow_query_log%';
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 1;

-- 查看慢查询
SELECT * FROM mysql.slow_log ORDER BY start_time DESC LIMIT 10;
```

### 查看查询执行计划

```sql
EXPLAIN SELECT * FROM contracts WHERE status = 'active';
```

### 查看连接状态

```sql
SHOW STATUS LIKE 'Threads_connected';
SHOW STATUS LIKE 'Max_used_connections';
```

### 锁等待分析

```sql
-- 查看当前事务
SELECT * FROM information_schema.INNODB_TRX;

-- 查看锁
SELECT * FROM information_schema.INNODB_LOCKS;
```
