# 安信合同管理系统 - 安全配置指南

本文档提供系统安全配置指南，确保系统安全运行。

## 1. 安全架构

### 1.1 安全目标

- 保密性：保护敏感数据不被未授权访问
- 完整性：确保数据不被未授权修改
- 可用性：确保系统在需要时可用

### 1.2 安全层级

```
┌─────────────────────────────────────┐
│          应用安全                    │
│  JWT认证 / 权限控制 / 输入验证       │
├─────────────────────────────────────┤
│          数据安全                    │
│  密码加密 / 完整性校验 / 审计日志    │
├─────────────────────────────────────┤
│          传输安全                    │
│  HTTPS / TLS 加密                   │
├─────────────────────────────────────┤
│          基础设施安全                │
│  防火墙 / 访问控制 / 安全更新        │
└─────────────────────────────────────┘
```

## 2. 认证安全

### 2.1 JWT配置

```env
# 生产环境必须修改
SECRET_KEY=<生成的安全密钥>

# Token过期时间
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

生成安全密钥：

```bash
openssl rand -base64 32
```

### 2.2 登录安全

系统内置登录保护机制：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 最大登录尝试 | 5次 | 超过后锁定 |
| 锁定时长 | 15分钟 | 自动解锁 |
| 失败记录重置 | 30分钟 | 自动清除 |

### 2.3 密码策略

- 密码加密：bcrypt（不可逆）
- 密码杂凑：SHA-256（前端验证）
- 最小长度：6位

## 4. 数据安全

### 4.1 敏感数据保护

| 数据类型 | 保护措施 |
|----------|----------|
| 用户密码 | bcrypt加密 |
| 密码杂凑 | SHA-256 |
| 完整性哈希 | SHA-256 |
| 审计日志 | 完整记录 |

### 4.2 数据库安全

```sql
-- 创建专用应用用户
CREATE USER 'app_user'@'localhost' IDENTIFIED BY '强密码';
GRANT SELECT, INSERT, UPDATE, DELETE ON contract_manage.* TO 'app_user'@'localhost';
FLUSH PRIVILEGES;

-- 删除root远程登录
DELETE FROM mysql.user WHERE User='root' AND Host='%';
FLUSH PRIVILEGES;
```

## 6. 网络安全

### 6.1 防火墙配置

```bash
# 只开放必要端口
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP
ufw allow 443/tcp   # HTTPS
ufw enable
```

### 6.2 HTTPS配置

```nginx
server {
    listen 443 ssl;
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
}
```

### 6.3 Nginx安全头

```nginx
add_header X-Frame-Options "SAMEORIGIN";
add_header X-Content-Type-Options "nosniff";
add_header X-XSS-Protection "1; mode=block";
```

## 7. 服务器安全

### 7.1 系统安全

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 配置防火墙
sudo ufw enable

# 禁用root登录
sudo passwd -l root

# 配置SSH密钥登录
```

### 7.2 MySQL安全

```bash
# 运行安全安装向导
sudo mysql_secure_installation

# 检查用户权限
SELECT user, host FROM mysql.user;
```

## 8. 日志与监控

### 8.1 审计日志

系统自动记录：

| 日志类型 | 内容 |
|----------|------|
| 登录日志 | 用户登录/登出 |
| 操作日志 | CRUD操作 |
| 访问日志 | API请求 |
| 错误日志 | 系统错误 |

### 8.2 日志保留

- 操作日志：180天
- 登录日志：180天
- 错误日志：90天

## 10. 应急响应

### 10.1 安全事件处理流程

1. **发现**：识别安全事件
2. **隔离**：切断受影响系统
3. **评估**：确定影响范围
4. **修复**：修复漏洞
5. **恢复**：恢复系统运行
6. **总结**：编写事件报告

### 10.2 常用命令

```bash
# 查看登录失败记录
tail -f /var/log/auth.log | grep failed

# 查看当前连接
netstat -tn | grep :8000

# 查看异常访问
tail -f /var/log/nginx/access.log
```
