# 安信合同管理系统 - 运维操作手册

本文档提供常见运维操作的详细步骤。

## 1. 服务管理

### 1.1 后端服务

```bash
# 查看服务状态
systemctl status contract-manage

# 启动服务
systemctl start contract-manage

# 停止服务
systemctl stop contract-manage

# 重启服务
systemctl restart contract-manage

# 查看日志
journalctl -u contract-manage -f
```

### 1.2 Docker服务

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启单个服务
docker-compose restart backend

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f backend
```

## 2. 数据库操作

### 2.1 连接数据库

```bash
# 命令行连接
mysql -u root -p contract_manage

# Docker内连接
docker exec -it contract_mysql mysql -u root -p
```

### 2.2 常用SQL

```sql
-- 查看表大小
SELECT table_name, ROUND(data_length/1024/1024, 2) AS 'MB' 
FROM information_schema.tables 
WHERE table_schema = 'contract_manage';

-- 查看慢查询
SELECT * FROM mysql.slow_log ORDER BY start_time DESC LIMIT 10;

-- 查看连接数
SHOW STATUS LIKE 'Threads_connected';

-- 清理测试数据
DELETE FROM contracts WHERE contract_no LIKE 'TEST-%';
```

## 3. 日志管理

### 3.1 日志位置

| 日志类型 | 位置 |
|----------|------|
| 应用日志 | stdout / var/log/contract/ |
| 访问日志 | Nginx access.log |
| 错误日志 | Nginx error.log |
| 数据库日志 | /var/log/mysql/ |

### 3.2 日志分析

```bash
# 查看错误日志
tail -100 /var/log/nginx/error.log

# 实时查看应用日志
tail -f /var/log/contract/app.log

# 统计错误数量
grep -c ERROR /var/log/contract/app.log

# 查看最近1小时日志
find /var/log/contract -mtime -1 -exec ls -l {} \;
```

## 4. 性能优化

### 4.1 数据库优化

```sql
-- 优化表
OPTIMIZE TABLE contracts;

-- 分析表
ANALYZE TABLE contracts;

-- 查看索引
SHOW INDEX FROM contracts;

-- 查看查询计划
EXPLAIN SELECT * FROM contracts WHERE status = 'active';
```

### 4.2 缓存清理

```bash
# 清理系统缓存
sync && echo 3 > /proc/sys/vm/drop_caches

# 清理Docker缓存
docker system prune -a
```

## 5. 用户管理

### 5.1 创建用户

```bash
# 通过API创建
curl -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "newuser",
    "password": "password123",
    "full_name": "新用户",
    "role": "sales"
  }'
```

### 5.2 重置密码

```sql
-- 通过数据库重置（需要知道bcrypt hash）
UPDATE users SET hashed_password='$2a$10$xxx' WHERE username='admin';
```

### 5.3 禁用用户

```sql
UPDATE users SET account_status='disabled' WHERE username='user1';
```

## 6. 数据迁移

### 6.1 导出数据

```bash
# 导出整个数据库
mysqldump -u root -p contract_manage > contract_manage.sql

# 导出指定表
mysqldump -u root -p contract_manage contracts customers > tables.sql

# 导出结构
mysqldump -u root -p -d contract_manage > schema.sql
```

### 6.2 导入数据

```bash
# 导入整个数据库
mysql -u root -p contract_manage < contract_manage.sql

# 导入到新数据库
mysql -u root -p -e "CREATE DATABASE new_contract"
mysql -u root -p new_contract < contract_manage.sql
```

## 7. 磁盘管理

### 7.1 检查磁盘

```bash
# 查看磁盘使用
df -h

# 查看目录大小
du -sh /var/www/contract/*

# 查找大文件
find / -type f -size +100M
```

### 7.2 清理日志

```bash
# 清理Nginx日志
> /var/log/nginx/access.log
> /var/log/nginx/error.log

# 清理系统日志
journalctl --vacuum-time=7d

# 清理Docker日志
truncate -s 0 /var/lib/docker/containers/*/*.log
```

## 9. 服务健康检查

### 9.1 常用检查

```bash
# 后端健康检查
curl http://localhost:8000/health

# 数据库连接
mysql -u root -p -e "SELECT 1"

# Nginx状态
nginx -t

# 磁盘IO
iostat -x 1 1
```

### 9.2 自动检查脚本

```bash
#!/bin/bash
# health-check.sh

# 检查后端
curl -sf http://localhost:8000/health || echo "Backend: DOWN"

# 检查数据库
mysql -u root -p -e "SELECT 1" || echo "Database: DOWN"

# 检查磁盘
df -h | awk '$5 > 80 {print "Disk: "$5}'
```

## 10. 常用运维命令速查

```bash
# 服务相关
systemctl status contract-manage
systemctl restart contract-manage
journalctl -u contract-manage -n 100

# 数据库
mysql -u root -p contract_manage
mysqldump -u root -p contract_manage > backup.sql

# 日志
tail -f /var/log/contract/app.log
grep ERROR /var/log/contract/app.log

# 网络
netstat -tlnp | grep -E "8000|3306"
lsof -i :8000

# 进程
ps aux | grep main
top -p $(pgrep main)
```
