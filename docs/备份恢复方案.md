## 2. 备份方案

### 2.1 MySQL备份脚本

```bash
#!/bin/bash
# backup.sh - 数据库备份脚本

# 配置
BACKUP_DIR="/backup/mysql"
DATE=$(date +%Y%m%d)
DB_NAME="contract_manage"
DB_USER="root"
DB_PASS="your_password"

# 创建备份目录
mkdir -p ${BACKUP_DIR}

# 全量备份
mysqldump -u${DB_USER} -p${DB_PASS} \
  --single-transaction \
  --routines \
  --triggers \
  --events \
  ${DB_NAME} | gzip > ${BACKUP_DIR}/full_${DATE}.sql.gz

# 删除30天前的备份
find ${BACKUP_DIR} -name "full_*.sql.gz" -mtime +30 -delete

echo "Backup completed: ${DATE}"
```

### 2.2 定时任务配置

```bash
# crontab -e
# 每日凌晨2点全量备份
0 2 * * * /opt/backup/backup.sh >> /var/log/backup.log 2>&1

# 每小时增量备份
0 * * * * /opt/backup/incremental.sh >> /var/log/backup.log 2>&1
```

### 2.3 Docker环境备份

```bash
# 备份数据库容器
docker exec contract_mysql mysqldump -u root -p contract_manage > backup_$(date +%Y%m%d).sql

# 备份配置
tar -czf config_$(date +%Y%m%d).tar.gz .env

# 备份上传文件
tar -czf uploads_$(date +%Y%m%d).tar.gz uploads/
```

## 4. 数据恢复

### 4.1 恢复场景

| 场景 | 恢复内容 | 恢复时间 |
|------|----------|----------|
| 误删除数据 | 表数据 | 10分钟 |
| 表损坏 | 单个表 | 5分钟 |
| 数据库损坏 | 整个库 | 30分钟 |
| 服务器迁移 | 全部数据 | 1小时 |

### 4.2 恢复命令

```bash
# 完全恢复
zcat /backup/mysql/full_20240101.sql.gz | mysql -u root -p contract_manage

# 从容器恢复
docker exec -i contract_mysql mysql -u root -p contract_manage < backup.sql

# 恢复到指定时间点
mysqlbinlog --stop-datetime="2024-01-01 10:00:00" mysql-bin.000001 | mysql -u root -p
```

### 4.3 部分恢复

```bash
# 只恢复单个表
mysqldump -u root -p contract_manage contracts > contracts.sql
mysql -u root -p contract_manage -e "DROP TABLE contracts"
mysql -u root -p contract_manage < contracts.sql
```

## 5. 异地备份

### 5.1 方案设计

```
本地备份服务器 ──► 云存储/远程服务器
     │                    │
  日志备份            加密传输
```

### 5.2 Rsync同步

```bash
# 同步到远程服务器
rsync -avz --progress /backup/mysql/ user@remote-server:/backup/mysql/

# 删除远程过期文件
ssh user@remote-server "find /backup/mysql -mtime +30 -delete"
```

### 5.3 云存储备份

```bash
# 阿里云OSS
aliyun oss cp /backup/mysql/full_20240101.sql.gz oss://bucket-name/

# AWS S3
aws s3 cp /backup/mysql/full_20240101.sql.gz s3://bucket-name/
```

## 6. 灾难恢复

### 6.1 恢复流程

```
1. 评估损失
   ↓
2. 确定恢复点
   ↓
3. 准备恢复环境
   ↓
4. 执行恢复
   ↓
5. 验证数据
   ↓
6. 业务恢复
```

### 6.2 RTO/RPO目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| RTO | 4小时 | 恢复时间目标 |
| RPO | 1小时 | 恢复点目标 |

### 6.3 恢复演练

每季度进行一次恢复演练：

1. 在测试环境恢复备份
2. 验证数据完整性
3. 测试应用功能
4. 记录恢复时间
5. 优化恢复流程

## 7. 备份存储

### 7.1 存储规划

| 备份类型 | 存储位置 | 保留时间 |
|----------|----------|----------|
| 日备份 | 本地磁盘 | 7天 |
| 周备份 | 本地磁盘 | 30天 |
| 月备份 | 远程存储 | 1年 |

### 7.2 存储要求

- 本地备份：至少2倍日备份大小
- 远程备份：至少100GB可用空间
- 加密存储：敏感数据加密
