## 2. 测试类型

### 2.1 单元测试

**目标**：验证单个函数/方法的功能

**工具**：Go testing

```bash
# 运行单元测试
go test -v ./services/

# 运行单个测试
go test -v -run TestUserService ./services/
```

**覆盖范围**：
- 用户服务
- 客户管理服务
- 合同管理服务
- 审批服务
- 工作流服务

### 2.2 集成测试

**目标**：验证多个模块间的协作

**工具**：Go testing + MySQL

```bash
# 运行集成测试
go test -v -run TestFullApprovalWorkflow ./services/
```

**测试场景**：
- 完整审批流程
- 客户创建流程
- 合同生命周期
- 权限控制流程

### 2.3 端到端测试

**目标**：验证完整业务流程

**工具**：curl / Postman

**测试流程**：
1. 用户登录 → 获取Token
2. 创建客户 → 验证返回
3. 创建合同 → 提交审批
4. 审批通过 → 验证状态

## 5. 自动化测试

### 5.1 CI/CD集成

```yaml
# .github/workflows/test.yml
name: Test
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run tests
        run: go test -v ./...
      - name: Run integration tests
        run: go test -v -run TestFullApprovalWorkflow ./services/
```

### 5.2 测试脚本

```bash
#!/bin/bash
# run-tests.sh

echo "Running unit tests..."
go test -v -cover ./services/

echo "Running integration tests..."
go test -v -run "TestFullApprovalWorkflow|TestApprovalWorkflow" ./services/

echo "All tests completed!"
```
