# 安信合同管理系统 - 版本升级指南

本文档提供系统版本升级的步骤和注意事项。

## 1. 升级概述

### 1.1 版本命名规则

```
主版本号.次版本号.修订号
v1.0.0
  │  │  └─ 修复问题
  │  └────── 新功能（向后兼容）
  └────────── 不兼容变更
```

### 1.2 升级类型

| 类型 | 说明 | 风险 |
|------|------|------|
| 小版本升级 | 修订号变更 | 低 |
| 中版本升级 | 次版本号变更 | 中 |
| 大版本升级 | 主版本号变更 | 高 |

---

## 2. 升级前准备

### 2.1 备份

```bash
# 备份数据库
mysqldump -u root -p contract_manage > backup_$(date +%Y%m%d).sql

# 备份配置文件
cp .env .env.backup

# 备份上传文件
tar -czf uploads_backup.tar.gz uploads/
```

### 2.2 环境检查

```bash
# 检查当前版本
cat version.txt

# 检查Go版本
go version

# 检查MySQL版本
mysql -u root -p -e "SELECT VERSION()"

# 检查Node版本
node -v
```

### 2.3 依赖检查

```bash
# 检查Go依赖
go mod tidy
go mod verify

# 检查前端依赖
cd frontend
npm install
npm audit
```

## 3. 升级步骤

### 3.1 小版本升级 (1.0.x → 1.0.y)

```bash
# 拉取最新代码
git pull

# 更新依赖
go mod download

# 重启服务
systemctl restart contract-manage
# 或
docker-compose restart backend
```

### 3.2 中版本升级 (1.x.y → 1.(x+1).0)

```bash
# 拉取最新代码
git pull

# 更新依赖
go mod tidy
go mod download

# 运行数据库迁移
go run scripts/migrate.go

# 重启服务
systemctl restart contract-manage
```

### 3.3 大版本升级 (x.0.0 → (x+1).0.0)

```bash
# 拉取最新代码
git pull

# 备份数据库（重要！）
mysqldump -u root -p contract_manage > full_backup.sql

# 更新依赖
go mod tidy

# 检查breaking changes
cat CHANGELOG.md

# 运行迁移脚本
go run scripts/migrate.go

# 检查配置变更
diff .env.example .env

# 测试环境验证
# 1. 在测试环境运行
# 2. 运行测试用例
go test ./...

# 重启服务
systemctl restart contract-manage
```

## 4. 数据库迁移

### 4.1 迁移脚本

```go
// migrations/migrate.go
func main() {
    // 检查并创建新表
    db.AutoMigrate(&NewTable{})
    
    // 数据迁移
    db.Exec("ALTER TABLE contracts ADD COLUMN new_field VARCHAR(100)")
}
```

### 4.2 回滚脚本

```bash
# 如果需要回滚
mysql -u root -p contract_manage < backup.sql
```

## 5. 升级验证

### 5.1 功能验证

```bash
# 检查服务状态
curl http://localhost:8000/health

# 测试登录
curl -X POST http://localhost:8000/api/auth/login

# 运行测试
go test -v ./...
```

### 5.2 数据验证

```sql
-- 检查数据完整性
SELECT COUNT(*) FROM contracts;
SELECT COUNT(*) FROM customers;
SELECT COUNT(*) FROM users;

-- 检查外键关系
SELECT * FROM contracts WHERE customer_id NOT IN (SELECT id FROM customers);
```

## 6. 常见问题

### 6.1 升级失败

**问题**：数据库迁移失败

**解决**：
1. 检查错误日志
2. 手动执行SQL
3. 回滚到备份

### 6.2 兼容性问题

**问题**：配置文件不兼容

**解决**：
1. 对比配置差异
2. 更新配置项
3. 重启服务

### 6.3 依赖冲突

**问题**：Go依赖冲突

**解决**：
1. 删除go.sum
2. 重新下载依赖
3. 验证模块
