# 安信合同管理系统 - 业务逻辑分析报告

---

## 一、系统架构概览

### 技术栈
| 层级 | 技术选型 |
|------|----------|
| 后端 | Go + Gin + GORM + MySQL |
| 前端 | Vue3 + Element Plus + Pinia + Vite |
| 认证 | JWT Token |
| 部署 | Docker Compose |

### 核心模块
| 模块 | 路由前缀 | 功能描述 |
|------|----------|----------|
| 用户认证模块 | `/api/auth` | 注册、登录、用户管理 |
| 客户管理模块 | `/api/customers` | 客户/供应商信息管理 |
| 合同管理模块 | `/api/contracts` | 合同CRUD、状态管理、生命周期跟踪 |
| 审批管理模块 | `/api/approvals` | 合同审批、状态变更审批 |
| 工作流模块 | `/api/workflow` | 多级审批流程 |
| 审计日志模块 | `/api/audit-logs` | 操作日志记录与查询 |
| 统计报表模块 | `/api/statistics` | 数据统计与分析 |

### 一键部署
系统提供 `deploy.sh` 脚本，支持自动检测并安装环境组件：

```bash
./deploy.sh start
```

支持的操作系统：CentOS/RHEL/Rocky/AlmaLinux/Ubuntu/Debian

               │                   │                   │

        ┌─────────────┐
        │  待审批     │
        │ (pending)   │
        └──────┬──────┘
               │

        ┌─────────────┐

        └──────┬──────┘
               │
          完成执行 │
               ▼
        ┌─────────────┐
        │  已完成     │

               │

        │  已归档     │
        │ (archived)  │
        └─────────────┘
```

**简化后的合同状态流转：**

```
草稿(draft) → 待审批(pending) → 已生效(active) → 已完成(completed) → 已归档(archived)
                   ↓
              已终止(terminated)
```

### 状态说明
| 状态值 | 状态名称 | 说明 |
|--------|----------|------|
| `draft` | 草稿 | 合同初始状态 |
| `pending` | 待审批 | 已提交等待审批 |
| `active` | 已生效 | 合同正式生效 |
| `completed` | 已完成 | 合同履行完毕 |
| `terminated` | 已终止 | 合同终止 |
| `archived` | 已归档 | 合同归档保存 |

           │                 │                 │

           │                 │                 │
           │                 │       ┌─────────┴─────────┐
           │                 │       │                   │
           │                 │   需要审批           不需要审批
           │                 │   (2种状态)          (其他状态)
           │                 │       │                   │
           ▼                 │       ▼                   ▼

           │                 │
           └─────────────────┘
```

### 需要审批的状态变更
| 状态 | 需要审批 | 说明 |
|------|----------|------|
| `archived` (归档) | ✅ 是 | 需要管理员审批 |
| `terminated` (终止) | ✅ 是 | 需要管理员审批 |
| 其他状态 | ❌ 否 | 直接变更，无需审批 |

### 状态转换规则
| 当前状态 | 目标状态 | 是否需要审批 | 前置条件 |
|----------|----------|--------------|----------|
| draft | pending | 否 | 提交审批 |
| pending | active | 否 | 审批通过 |
| active | completed | 否 | 执行完成 |
| any | archived | **是** | 归档操作 |
| any | terminated | **是** | 终止操作 |

## 十一、关键业务规则

### 1. 合同编号生成规则
```
格式：CT + YYYYMM + 4位序号
示例：CT2024010001
说明：CT + 年月 + 当月序号（从0001开始递增）
```

### 2. 权限控制规则
| 角色 | 权限描述 |
|------|----------|
| `admin` | 全部权限，可管理系统所有功能 |
| `manager` | 可审批草稿合同，管理本部门合同 |
| `user` | 只能操作自己创建的合同 |
| `audit_admin` | 只能查看审计日志，不能修改数据 |

### 3. 文档管理规则
- 支持 `.docx` 格式合同文档上传
- 自动解析合同关键信息（合同编号、金额、日期等）
- 文件大小限制：10MB
- 支持文档版本管理

### 4. 到期提醒规则
- 可配置提前天数（默认30天）
- 支持自定义提醒类型
- 可发送提醒通知
