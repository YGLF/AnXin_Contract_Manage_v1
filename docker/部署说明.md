# 安信合同管理系统 - Docker 部署指南

## 系统概述

安信合同管理系统是一套基于 Go + Gin + MySQL（后端）和 Vue3 + Element Plus（前端）的智能合同管理解决方案。

### 功能模块

| 模块 | 说明 |
|------|------|
| 用户权限管理 | 用户注册、登录、角色管理、细粒度权限控制 |
| 客户/供应商管理 | 客户信息管理、分类、信用等级 |
| 合同管理 | 合同信息管理、状态跟踪、归档管理 |
| 审批流程 | 三级审批、状态变更审批、工作流 |
| 通知提醒 | 到期提醒、审批提醒、待办事项 |
| 审计日志 | 操作记录、审计追踪 |
| 加密服务 | HSM/SM4/AES 加密接口 |

### 技术栈

| 组件 | 技术 |
|------|------|
| 后端 | Go 1.21 + Gin + GORM |
| 前端 | Vue 3 + Element Plus + Vite |
| 数据库 | MySQL 8.0 |
| Web服务器 | Nginx |

## 环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| 操作系统 | CentOS 7.x | 仅支持CentOS 7 |

## 快速部署

### 步骤1: 上传项目

将整个项目目录上传到服务器的 `/opt/` 目录：

```bash
cd /opt
# 通过FTP/SFTP上传项目后
ls -la AnXin_Contract_Manage_encrytp_v1.1/
```

### 步骤2: 执行部署

```bash
cd /opt/AnXin_Contract_Manage_encrytp_v1.1
chmod +x deploy.sh
./deploy.sh
```

部署过程会自动完成：
- [x] 安装Docker 24.x
- [x] 安装Docker Compose
- [x] 配置镜像加速器
- [x] 生成安全配置文件
- [x] 构建并启动所有容器
- [x] 配置定时备份

### 步骤3: 访问系统

部署完成后，访问：

- **前端**: http://服务器IP/
- **后端API**: http://服务器IP:8000

默认账号（首次登录后请立即修改密码）：
- 超级管理员: admin / (部署时生成的随机密码)
- 审计管理员: auditadmin / (部署时生成的随机密码)

## 备份与恢复

### 自动备份

系统已配置每天凌晨3点自动备份，备份文件保存在 `./backups/` 目录，保留7天。

### 手动备份

```bash
./backup.sh
```

### 恢复数据库

```bash
# 停止服务
docker-compose down

# 恢复数据（将backup文件复制到容器内）
docker cp backup.sql contract_mysql:/tmp/restore.sql

# 进入容器恢复
docker exec -it contract_mysql mysql -ucontract_user -p contract_manage < /tmp/restore.sql

# 重启服务
docker-compose up -d
```

## 故障排查

### 常见问题

#### 1. Docker安装失败

```bash
# 检查系统版本
cat /etc/centos-release

# 手动安装Docker
yum update
yum install -y docker-ce-24.0.7 docker-ce-cli-24.0.7
```

#### 2. 端口被占用

```bash
# 查看端口占用
netstat -tuln | grep -E '3306|8000|80'

# 终止占用进程
fuser -k 3306/tcp
fuser -k 8000/tcp
fuser -k 80/tcp
```

#### 3. 容器启动失败

```bash
# 查看容器日志
docker logs contract_mysql
docker logs contract_backend
docker logs contract_frontend

# 检查Docker状态
docker ps -a
docker-compose logs
```

#### 4. 数据库连接失败

```bash
# 检查MySQL容器状态
docker exec -it contract_mysql mysqladmin ping

# 进入MySQL容器
docker exec -it contract_mysql mysql -ucontract_user -p

# 检查网络连通性
docker network ls
docker network inspect contract-manage_contract_network
```

#### 5. 前端无法访问

```bash
# 检查Nginx状态
docker logs contract_frontend

# 检查文件是否构建成功
docker exec contract_frontend ls -la /usr/share/nginx/html
```

### 完全重置

如需完全重新部署：

```bash
# 停止并删除所有容器
docker-compose down -v

# 清理Docker缓存
docker system prune -af

# 重新部署
./deploy.sh
```

## 安全建议

1. **修改默认密码**: 首次部署后及时修改管理员密码
2. **配置防火墙**: 生产环境建议只开放80和8000端口
3. **定期备份**: 定期检查备份文件是否正常
4. **更新Docker**: 定期更新Docker到最新稳定版本
5. **日志监控**: 定期检查日志文件，及时发现异常

```bash

  -H "Authorization: Bearer <token>"
```
